# Contribuer au Projet

Nous sommes ravis que vous envisagiez de contribuer à notre projet ! Toute contribution, petite ou grande, est appréciée.
Voici quelques lignes directrices que nous vous demandons de suivre pour maintenir notre communauté accueillante et productive.

## Comment Contribuer ?

1. **Signaler des Bugs :** Si vous trouvez un bug, merci de vérifier si celui-ci a déjà été signalé. Si ce n'est pas le cas,
ouvrez un nouveau ticket dans notre système de suivi des problèmes. Veuillez fournir un rapport de bug clair et détaillé.

2. **Proposer des Modifications :** Si vous souhaitez apporter des modifications au code, merci de discuter de ces changements
avant de commencer à travailler. Cela nous aide à comprendre votre vision et à déterminer si elle s'intègre bien dans le projet.
Les propositions peuvent être soumises sous forme d'issues.

3. **Pull Requests :** Pour toute modification directe, veuillez soumettre une pull request. Assurez-vous de :
    - Créer une branche spécifique pour vos modifications.
    - Suivre les standards de codage du projet.
    - Inclure des tests pertinents, si applicable.
    - Documenter toute nouvelle fonctionnalité ou paramètre important.

## Normes de Codage

Nous suivons des standards de codage précis pour maintenir la qualité et la lisibilité du code :
- Utilisez des indentations cohérentes.
- Commentez votre code là où c'est nécessaire.
- Suivez les conventions de nommage en vigueur dans le projet.

## Tester

Avant de soumettre votre pull request, assurez-vous que votre code passe tous les tests existants et, si vous introduisez
une nouvelle fonctionnalité, ajoutez des tests pour couvrir cette fonctionnalité.

## Engagez-vous de Manière Respectueuse

Notre projet adhère à un code de conduite, et nous attendons de tous les contributeurs qu'ils l'offrent et en respectent
l'esprit. Une interaction respectueuse et constructive aide à maintenir une communauté saine et inclusive.
